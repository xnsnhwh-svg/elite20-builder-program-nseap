# Vision

Elite20 Phase II should be designed as a Builder Operating System.

The system should help students, teachers, and agents work together to rebuild an AI-native course as a reusable educational product.

## Core Principle

Every Challenge should become a Task Skill or Task Agent.

This means a Challenge is not just a message or homework description. It should be:

- reusable
- reviewable
- executable
- documentable
- improvable

## What We Build First

We do not start with a full LMS.

We first build the workflow that makes the course alive:

```text
Challenge published
-> Student builds with AI
-> Work submitted to GitHub
-> Peer review happens
-> Agent review happens
-> Instructor confirms
-> Good work becomes knowledge
-> Knowledge improves the next Challenge
```

## Reference From Phase I

Phase I is a reference sample, not the target system itself.

The useful patterns from Phase I are:

- Challenge-driven learning
- public sharing and mutual help
- AI logs and reflection
- GitHub-based artifacts
- automatic review attempts such as C4A
- project-based final delivery

Phase II should standardize these patterns into templates, workflows, and agents.
