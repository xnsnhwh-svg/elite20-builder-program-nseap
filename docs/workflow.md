# Builder Workflow

## Main Workflow

```text
Problem / goal
-> Challenge design
-> Challenge release
-> Student planning
-> AI-assisted building
-> GitHub submission
-> Self check
-> Peer review
-> Agent review
-> Instructor review
-> Merge / archive
-> Documentation
-> Release
-> Knowledge capture
```

## Roles

| Role | Responsibility |
|---|---|
| Instructor | defines goals, confirms final standards, makes final decisions |
| Builder | completes Challenges and improves the system |
| Peer Reviewer | reviews classmates' submissions and gives useful suggestions |
| Project Manager Agent | tracks tasks, deadlines, and review status |
| Coding Coach Agent | helps students complete and improve submissions |
| Evaluation Agent | checks submissions against the Rubric |

## GitHub Usage

GitHub is the formal delivery space.

Use GitHub for:

- Challenge definitions
- Issues
- Pull requests
- code
- documents
- AI logs
- review records
- release notes

Do not use GitHub as a chat room. Daily discussion can stay in WeChat, Feishu, or Xuexitong.
